Conoscenze e abilità da conseguire

Al termine del corso lo studente:
  • è in grado di impostare processi di sviluppo del software cooperativo basati sia su approcci agili (in particolare SCRUM) sia su approcci basati su modelli espressi in UML2 oppure espressi mediante meta-modelli custom, domain specific;
  • è in grado di valutare criticamente il ruolo e l'uso di UML2 come linguaggio di modellazione nella costruzione di sistemi distribuiti eterogeni
  • e' in grado di sviluppare sistemi software in modo technology-independent attraverso opportune formalizazioni dell'analisi dei requisiti e dell'analisi del problema con particolare riferinento al settore oggi noto come Internet of Things
  • è in grado di affrontare il progetto e la costruzione di piattaforme custom di supporto allo sviluppo di sistemi software proattivi/reattivi con particolare riferimento ai modelli message-passing e event.driven
  • è in grado di progettare e costruire (in ambiente Eclipse) ambienti di sviluppo custom (Software Factories) capaci di generazione automatica di codice in accordo all'approccio Model Driven Software Development (MDSDtt>)
  • è in grado di costruire dispositivi a basso costo tipici dell'Internet of Things (IOT) per lo svliuppo di case-study concreti basati su dispositici quali Raspberry, Arduino e Android con particolare riferimento a robot differential drive (DDR) custom
  • è in grado di applicare i concetti, i dispostivi, e gli strunmenti sviluppati in modo concreto ed operativo durante il laboratorio nello svluppo di applicazioni IOT che utilizzano DDR singoli o coordinati come sistemi che operano sulla base di dati sensoriali

From the methological/conceptual point of view:

  • Understand and manage methodologies and tools to make explicit knowldegde/information usually implicit in code-based software production processes, with particular attention to the phases of analysis, design and testing.
  • Understand the role of Requirement analysis and its practical impact on software pruduction (Domain model specification, Unit-Testing Plans, Work plan)
  • Understand the impact of a given Software Technolgy on software analysis, design and implementation
  • Understand the role of Problem analysis with particular reference to the definition of the logic architecture of a software system, on the evaluation of the abstraction gap between an application (domain) and the implementation technology, on the production of Integration-Testing Plans, on the evalutation of risks and d on the definition of the work-plans.
  • Learn how to overcome the limits of a technology-based software development approach via cooperative work based on logic architectures and MDSD (Model Driven software Development), able to promote collaboration between development, operations and testing (a graceful introduction to DevOps)
  • Learn how to automatize software construction by exploiting MDSD

From the practical point of view:

  • Define test plans and automate testing with JUnit and Gradle
  • Experiment with the SCRUM framework
  • Design and develop software systems according to an evoltive process of piecemeal growth along three main architectural dimentions, by intergrating model driven approces with SCRUM based methodology.
  • Learn how to deal with event-driven programming and with reactive programmimg
  • Learn how to build a software factory around domain specific languages by using the Xtext technology
  • Design and build distributed, heterogenous systems with particular reference to the domain of Internet of Things and to concrete platforms such as Android, Arduino, RaspberryPi.
  • Design and build control software for "intelligent devices" (our in-hous-made robots)

Laboratory

The course is based on a strong interaction between theory and laboratory. The leraning activties will be organized with reference to a sequence of projects to be devoloped in the lab.

The aim of each proposed project is to face some specific methodology and/or aspect of software development, in incresing order of complexity. The activity performed in the lab related to a project is organized so to produce software that can be reused in the following projacts. Thus, course attendance is strongly reccomended.

Each project will be associated with a set of suggested readings, in order to establish a proper 'knowledge base' around the practical activities.

    Assessment

    The examination will performed in two-phases.

    The first phase starts by publishing a set of requirements and terminates with the production of a software prototype satisfying the given requirements, toghether with a project report. For an example of requirements, see TemaFinaleLMBO2017.html
    The first phase can be delevoped both in individual way or in a team of at most 3 persons.

    The second phase consists of an individual discussion of the work done, in order to evaluate personal skills and the background knowledge of each member of the team.
