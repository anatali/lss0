ButtonLedSystem based on GUI

VISION

  • Any project should start from a system logical architecture, defined in the problem analysis phase.
  • The definition of a logical architecture can be followed by the development of reusable software working as a framework.

GOALS

  • Show how the software built from the logical architecture introduced in BLSOOPFromLogicArch can be used to create a ButtonLed system with a button and a led implemented as GUI
-------

Requirements

User story: As a operator, I want to use a graphical user interface with a button named SWITCH and a pitcure representing a Led in the off state. When I press the SWITCH Button, the Led picture should show a Led on. When I press again the SWITCH Button the Led picture should show a Led off.

Towards distributed systems

Exchanging messages: IConnInteraction

The project it.unibo.noawtsupports defines a support for communications over the network based on two-way connected protocols (like TCP, UDP) or a serial line (the case of Arduino). It provides:
  1. A factory FactoryProtocol to create supports for a server or for a client (in the case of TCP/UDP) or for the user of a serial line.
  2. Each operation of the factory returns an object thet implements the interface IConnInteraction (defined in project it.unibo.interfaces) that can be used to send/receive information on the connection.
In this way the applicative software does not depend on the protocol used for communications. If we want to change a protocol, software modification is limited to the system configuration phase.
